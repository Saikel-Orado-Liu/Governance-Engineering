# Turn-Based Strategy — 需求规格 (GDD)

> 本文件为需求基准，Confirm Agent 基于此进行需求确认和歧义识别。

## 游戏概述

8x8 网格回合制策略对战，玩家 vs AI，类似简化版战棋游戏。

## 功能需求

### FR-01: 地图系统
- 8x8 网格，支持地形类型（平原/森林/山地/水域）
- 地形影响单位移动消耗

### FR-02: 单位系统
- 至少 4 种单位类型（战士/弓手/骑士/法师）
- 属性：HP、攻击力、攻击范围、移动力
- 单位间克制关系

### FR-03: 回合管理
- 玩家回合 → AI 回合循环
- 每回合单位可以移动 + 攻击（或技能）
- 移动范围高亮（基于移动力 + 地形）

### FR-04: AI 对手
- 简单行为树：选择目标 → 移动到攻击范围 → 攻击
- 优先攻击低血量单位

### FR-05: 战斗系统
- 伤害公式：ATK - DEF（最小 1 点）
- 攻击范围判定（近战 1 格，远程 2-3 格）

### FR-06: 存档系统
- Save/Load 完整游戏状态
- 序列化为 JSON

### FR-07: UI
- Canvas 渲染网格地图
- 单位信息面板
- 回合结束按钮
- 胜利/失败画面

## 非功能需求
- 游戏状态不可变更新
- 存档往返一致性（Save → Load → Save 等价于原 Save）
- 60 FPS 稳定渲染
- 模块间耦合通过接口而非具体实现

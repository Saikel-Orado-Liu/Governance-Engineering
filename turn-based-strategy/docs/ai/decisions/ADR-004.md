# ADR-004: 等距渲染架构 — IsometricRenderer 纯数学层 + AnimationManager RAF 动画

## 状态
已接受

## 背景
原型阶段的 MapRenderer 使用矩形网格渲染（640x640 Canvas，80px/格，fillRect 绘制矩形瓦片），单位渲染采用 Canvas 圆形+字母图标（16px 圆 + ctx.shadowBlur=6 发光）。为将视觉品质从"原型"提升至"类游戏"标准，需要实现等距菱形投影（isometric diamond projection）渲染管线，并添加平滑移动动画。

关键需求：
1. 网格从正投影改为等距菱形投影，瓦片宽高比 2:1
2. 单位图标从字母符号改为几何图形（盾/弓/V/五角星），消除"原型感"
3. 加入 RAF 驱动的平滑移动动画替代格子跳跃
4. 渲染管线与游戏逻辑层完全解耦

## 决策

### 1. 三层渲染架构分层

```
IsometricRenderer (src/render/) — 纯数学坐标变换层，零游戏实体依赖
  ↓ 坐标/绘制命令
MapRenderer (src/map/) — 游戏渲染层，持有 MapGrid 和 UnitManager 的消费者
  ↓ 渲染完成
Canvas + DOM UI
```

- **IsometricRenderer** 为纯函数模块（非类），导出 worldToScreen/screenToWorld/drawDiamondTile。零游戏实体依赖，可独立测试——纯线性代数问题，不应与游戏实体耦合
- **MapRenderer** 内部调用 IsometricRenderer 做坐标变换，公开 API 签名完全不变，仅内部渲染管线更改——main.ts 和 TurnManager 的调用代码无需修改
- **drawDiamondTile** 可被 MapRenderer 外部直接用于覆盖层（选中高亮、移动范围、攻击目标、技能范围、技能目标全部菱形绘制）

### 2. AnimationManager 独立层

- **AnimationManager** 为独立类，使用 RAF (requestAnimationFrame) 驱动 tick 循环
- **MapRenderer** 通过 `setAnimationManager(am)` 桥接：renderUnits 调用 am.getVisualPosition(unit) 获取插值位置；无动画时返回网格位置（退化为无动画渲染）
- **main.ts** 在单位移动后调用 `animationManager.startMoveAnimation(unit, fromRow, fromCol, toRow, toCol)` 触发动画
- 分离理由：动画逻辑不属于渲染器的职责；动画管理器按需启动/停止 RAF 循环（仅动画活跃时有开销），渲染器每一帧都需要渲染
- 动画完成时自动调用 unit.setPosition(toRow, toCol) 同步网格位置，确保逻辑层位置最终一致

### 3. 四种兵种的 Canvas 几何符号

| 兵种 | 符号 | 路径 |
|------|------|------|
| Warrior (0) | 盾形 | 半圆(8px) + 梯形底边 |
| Archer (1) | 弓形 | 圆弧(9px, ±0.4π) + 弦 |
| Knight (2) | V形(Chevron) | 两条直线 (r=9) |
| Mage (3) | 五角星 | 交替内外半径 (9/4px) |

替代了原型的圆形+字母首字——保留符号与兵种的映射关系但以图形化表达。

### 4. 深度排序（Painter's Algorithm）

- render() 中的瓦片绘制按 `row+col = sum` 分组，sum 从 0 到 GRID_SIZE*2-2 递增——先绘远后绘近
- renderUnits 按 `(row + col)` 升序排序单位，确保视觉正确的前后遮挡（行数大的角色覆盖行数小的角色）
- 选中高亮在 renderUnits 末尾绘制，作为独立覆盖层在所有单位之上

### 5. HP 条从单位下移到单位上

- 原型：单位圆下方 (cy+18)
- 现在：单位菱形上方 (cy-18)——避免与等距菱形图标视觉重叠

## 备选方案

| 方案 | 放弃理由 |
|------|---------|
| MapRenderer 内部直接实现等距计算 | 坐标变换 + 渲染耦合不利于复用；IsometricRenderer 零依赖可被 main.ts 直接用于 tooltip 点击检测 |
| AnimationManager 内置于 MapRenderer | MapRenderer 的 render() 是无副作用绘制，动画状态管理是正交职责；分离后 AnimationManager 由 main.ts 管理生命周期 |
| 使用精灵图/纹理渲染单位 | 不符合项目"零外部资源"约束，Canvas 几何路径直接绘制更轻量 |
| 使用 CSS transform 实现等距投影 | CSS transform 无法与 Canvas 高亮层（移动范围/攻击目标）坐标对齐——所有渲染必须在同一 Canvas 上下文中 |
| 使用第三方缓动库 | 仅需一个 easeInOutQuad 函数，内置即可，零依赖策略优于引入库 |

## 影响

- 正面：IsometricRenderer 纯函数可在 MapRenderer 之外复用——main.ts 的 tooltip 直接调用 screenToWorld 做点击检测
- 正面：AnimationManager 使用 RAF，仅在动画活跃时消耗 CPU，空闲时 RAF 自动暂停，不运行时无开销
- 正面：几何符号路径硬编码，无外部资源依赖，加载零延迟
- 正面：MapRenderer 公开 API 签名完全不变——main.ts 和 TurnManager 调用代码无需修改，仅渲染管线内部更换
- 约束：Canvas 画布从 640x640 矩形改为 640x360 宽屏等距视图
- 约束：等距原点 ORIGIN_X=320, ORIGIN_Y=52 硬编码，与 Canvas 尺寸和 TILE_W/TILE_H 形成隐含比例关系
- 约束：深度排序要求 renderUnits 在 render() 之后调用——瓦片背景层优先绘制，单位在瓦片之上叠加，单位间按 (row+col) 排序
- 约束：MapRenderer 移除了 ctx.shadowBlur 发光效果——几何符号路径绘制没有对应 shadow 语义，发光效果在等距视角下视觉上不再必要

# Task Board — 需求规格

> 本文件为需求基准，治理工程架构的 Confirm Agent 将基于此文件进行需求确认和歧义识别。

## 功能概述

一个全栈任务管理应用（类似 Trello 简化版），支持看板视图和列表视图。

## 功能需求

### FR-01: 任务 CRUD
### FR-02: 看板列管理（状态流转）
### FR-03: 任务拖拽排序
### FR-04: 任务标签与优先级
### FR-05: 数据持久化（SQLite）

## 非功能需求

- 前端需覆盖 loading / empty / error 三种异步状态
- 后端 API 需通过 OpenAPI 自动文档校验
- 安全性需通过 Bandit 扫描（零高危漏洞）

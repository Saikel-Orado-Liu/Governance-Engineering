# ADR-003: 原生 HTML5 DnD 替代 @dnd-kit 实现看板拖拽

## 状态

已接受

## 背景

第一阶段（ADR-002）选择 @dnd-kit 实现 Kanban 看板拖拽，采用四组件层级（KanbanBoard → KanbanColumn → KanbanCard + Toast）+ useTasks hook 的架构。实际运行中发现：

1. **Props drilling 深度大**：拖拽状态需穿越 Board → Column → Card 三层，每个中间组件转发不必要的 props
2. **Hooks 顺序 bug**：useTasks 内 useRef + useState + useCallback 的依赖链在 React 18 strict mode 下因纯函数组件重执行暴露出顺序敏感问题
3. **~30KB gzip 依赖**：三包 @dnd-kit 占用了不可忽略的 bundle 体积，而实际拖拽场景仅为跨列移动（无同列排序需求），未充分利用 @dnd-kit 的 Sortable 能力
4. **Toast 通知机制过度设计**：PATCH 失败场景仅需回滚 + 重新加载，无需 Toast 4 秒计时器组件

## 决策

1. **拖拽方案**：采用原生 HTML5 Drag and Drop API（`draggable` + `onDragStart`/`onDragOver`/`onDrop`），完全移除 @dnd-kit 三个依赖包
2. **组件扁平化**：单 Board.tsx 组件内联三列渲染 + 加载骨架屏 + 错误重试 + 乐观更新，消除 props drilling
3. **状态管理**：放弃 useTasks hook（4 组件 + hooks 模式），Board.tsx 内 useState + useCallback + useEffect 自包含管理
4. **错误处理**：`updateTaskStatus().catch(() => load())` — 失败时直接重新加载全量数据，放弃 Toast 通知组件
5. **CSS 优先级色条**：`Record<TaskPriority, string>` 查找表替代嵌套三元表达式
6. **删除组件**：KanbanBoard、KanbanColumn、KanbanCard、Toast 四个组件及 useTasks hook 全部删除

## 后果

正面：
- 消除三个外部依赖（@dnd-kit/core + sortable + utilities），bundle 体积减少 ~30KB gzip
- 组件层级从 4 层减少为 1 层，props 传递路径明晰
- 无 hooks 顺序风险（单组件内 useState/useCallback 顺序可控）
- 原生 API 零学习成本、零版本升级风险
- 加载骨架屏内联替代 Loading 文字，提升感知性能
- Record 查找表比嵌套三元更易维护扩展

负面：
- 原生 HTML5 DnD 在触屏设备上支持有限（缺少 touch events 自动处理），未来需补充 touch polyfill 或 pointer events 适配
- 放弃 @dnd-kit 意味着放弃动画过渡（drag overlay、drop animation），视觉反馈较粗糙
- 失败时全量 reload（vs 之前 per-task Map 回滚）在网络延迟大时可能导致 UI 闪烁
- 单 Board.tsx 文件约 150 行，功能进一步增加时需考虑拆分
